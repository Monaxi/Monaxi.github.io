Docker

Docker能做什么

将应用、依赖包打包到一个容器中，然后发布到流星的Linux机器上，也可以实现虚拟化。

应用场景：

  • Web应用的自动化打包和发布；
  • 自动化测试和持续集成、发布；
  • 在服务型环境中部署和调整数据库或其他后台应用；
  • 从头编译或扩展现有的OpenShift或Cloud Foundry平台搭建自己的PaaS环境；

常用的命令

  • 使用docker run命令在容器内运行一个应用程序（创建一个容器）
  • 使用-i、-t参数，让docker运行的容器实现对话能力

其中：

-i：允许对容器内的标准输入（STDIN）进行交互

-t：在新容器内指定一个伪终端或者终端

**root@1b658ae35706:/#**：进入了一个ubuntu:15.10系统的容器

  • 使用cat /proc/version查看当前系统版本信息
  • 使用ls查看当前目录下的文件列表
  • 使用exit命令或CTRL+D退出容器
  • 使用-d让容器在后台运行；

其中：

输出的长字符串叫做容器ID，对每个容器来说都是唯一的，可以通过容器ID查看对应的容器发生了什么；

  • 使用docker ps来查看容器是否在运行

其中：

CONTAINER ID: 容器 ID。

IMAGE: 使用的镜像。

COMMAND: 启动容器时运行的命令。

CREATED: 容器的创建时间。

STATUS: 容器状态。

状态有7种：

  • created（已创建）
  • restarting（重启中）
  • running 或 Up（运行中）
  • removing（迁移中）
  • paused（暂停）
  • exited（停止）
  • dead（死亡）

PORTS: 容器的端口信息和使用的连接类型（tcp\udp）。

NAMES: 自动分配的容器名称。

  • 使用-P将容器内部使用的网络端口随机映射到我们使用的主机上
